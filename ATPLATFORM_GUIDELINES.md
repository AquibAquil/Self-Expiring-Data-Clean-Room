# Atsign Platform SDK Guidelines

This document is the single source of truth for working with the Atsign Platform SDK in this project. Future LLM sessions MUST read this file before making changes.

## Core SDK References
- Primary SDK: `at_client` ^3.11.0 — https://pub.dev/packages/at_client
- Common utilities: `at_commons` (included with at_client) — https://pub.dev/packages/at_commons
- Flutter SDK: `at_client_flutter` ^1.0.1 — https://pub.dev/packages/at_client_flutter
- All Atsign packages: https://pub.dev/packages?q=publisher%3Aatsign.org&sort=updated
- Source code: https://github.com/atsign-foundation/at_client_sdk

## Architecture Principles
- Each person has their own Atsign (e.g. `@alice`, `@bob`). This is the core identity unit.
- Agents and applications may have a dedicated Atsign OR share an Atsign with their owner, using namespaces for isolation. Multiple agents/apps on one Atsign use different namespaces to keep data separate.
- Multiple instances of the same agent can run under a single Atsign (see Multi-Agent Coordination).
- IoT devices can have their own Atsign for autonomous operation, or be managed as keys under a person's Atsign.
- All data is end-to-end encrypted by default. Only the Atsign owner holds decryption keys.
- There is NO application backend. atServers act as personal data stores and rendezvous points.
- Use namespaces to separate application data (e.g. `key.myapp@atsign`).
- Use `AtKey` with `sharedWith` to share encrypted data with specific Atsigns.
- Use `AtKey` with `metadata.isPublic = true` only for intentionally public data.
- Do NOT store application data in local files. Store data as AtKeys so it synchronizes across all of the Atsign owner's devices.

## Atsign Validation — All Dart Code (CLI, Agents, and Flutter)
- Always use `at_commons` to validate every String used as an Atsign. e.g. `'@bob'.toAtsign()` ensures validity before use.
- API parameters that accept Atsigns can accept plain strings, but still validate. e.g. `atClient.getCurrentAtSign()?.toAtsign()`.
- `.toAtsign()` is case-sensitive. Anything else will fail at runtime. Provided by `at_commons` (included in `at_client` and `at_client_flutter`).

## Initialization Pattern (Dart CLI / Agents)
- Use `AtClientManager.getInstance().setCurrentAtSign(atSign, namespace, preferences)`.
- `AtClientPreference` with: `rootDomain: AtRootDomain.atsignDomain`, `hiveStoragePath`, `commitLogPath`, `namespace`.
- IMPORTANT: When running multiple agents/instances on the same machine, each MUST use a unique `hiveStoragePath` and `commitLogPath` (e.g. `Directory.systemTemp.createTempSync("agent_").path`). Sharing paths causes hive box collisions.
- For agent/CLI auth use `at_cli_commons` (`CLIBase`):
  ```dart
  final atClient = (await CLIBase.fromCommandLineArgs(args)).atClient;
  ```
  This handles arg parsing, atKeys loading, and PKAM auth in one line.
- APKAM enrollment for new devices is supported via `EnrollmentService` in `at_client`.

## Flutter App Requirements
- Use `at_client_flutter` (NOT `at_onboarding_flutter`, which is deprecated and unmaintained).
- Model auth/onboarding on the at_client_flutter example app: https://github.com/atsign-foundation/at_client_sdk/tree/trunk/packages/at_client_flutter/example
- When using `file_picker`, add ALL required platform permissions:
  - macOS entitlements: `user-selected.read-only` + `read-write` in both Debug and Release.
  - iOS Info.plist usage descriptions.
  - Android `READ_EXTERNAL_STORAGE`/`WRITE_EXTERNAL_STORAGE` manifest permissions.
- macOS: add `com.apple.security.network.client` and `com.apple.security.network.server` to both `DebugProfile.entitlements` and `Release.entitlements`.
- iOS: add `NSLocalNetworkUsageDescription` to Info.plist.

### Welcome / Auth Screen (must support all four workflows)
1. **Login from Keychain**: `KeychainStorage().getAllAtsigns()` → `AtSignSelectionDialog(existingAtSigns: list)` → `AtAuthRequest(atSign, atKeysIo: KeychainAtKeysIo())` → `PkamDialog` → `setCurrentAtSign()`.
2. **Onboard a New Atsign (Registrar flow)**: Follow `onboard()` in walkthrough.dart of the at_client_flutter example. Flow: `AtSignSelectionDialog.show(context)` → `RegistrarCramDialog.show(context, authRequest as AtOnboardingRequest, registrar: registrar)` → `CramDialog.show(context, request: authRequest, cramKey: cramKey)` → `_setupAtClient()`. Use `RegistrarService(registrarUrl: kRegistrarUrl, apiKey: kRegistrarApiKey)`, where both constants are read from `String.fromEnvironment(...)` so they can be overridden at build time via `--dart-define=REGISTRAR_API_KEY=...`. The default value in the code is Atsign's public demo key from the walkthrough — usable for local development, must be overridden for anything else.
3. **APKAM enrollment (new device)**: `AtSignSelectionDialog` → `ApkamActivationDialog.show(atSign, rootDomain, appName, deviceName, namespaces)` → wait for approval on an authorized device → `AtAuthRequest(atSign, atAuthKeys: enrollmentResponse.atAuthKeys)` → `PkamDialog` → `setCurrentAtSign()`.
4. **Login via .atKeys file**: `AtKeysFileDialog.show()` returns `FileAtKeysIo` → extract atSign from filename → `AtAuthRequest(atSign, atKeysIo: fileAtKeysIo)` → `PkamDialog(backupKeys: [KeychainAtKeysIo()])` → `setCurrentAtSign()`.

### Post-Auth Setup (all workflows end here)
- Get app support directory, create `AtClientPreference` with `rootDomain`, `namespace`, `hiveStoragePath`, `commitLogPath`.
- `AtClientManager.getInstance().setCurrentAtSign(atSign, namespace, prefs, atChops: response.atChops, atLookUp: response.atLookUp)`.

## Mandatory First-Run Atsign Gate
- Before any auth/onboarding/main UI, if `KeychainStorage().getAllAtsigns()` is empty, show a full-screen blocking gate explaining that Atsigns are required.
- Provide a "Get My Starter Pack" button opening https://my.atsign.com/starterpack_app and a "Continue" button to proceed to the auth screen.
- Persist a local flag only after at least one Atsign exists on the device.

## Communication Patterns

### 1. Async Communication (fire-and-forget)
- `atClient.put()` stores data for another Atsign; recipient retrieves with `get()` whenever they come online.
- Defaults to local secondary with later sync. Force remote with `putRequestOptions: PutRequestOptions()..useRemoteAtServer = true`.
- For `get()`: `GetRequestOptions()..useRemoteAtServer = true` or `..bypassCache = true`.
- Docs: https://pub.dev/documentation/at_client/latest/at_client/AtClient/put.html

### 2. Sync Communication (delivery confirmation)
- `notificationService.notify(NotificationParams.forUpdate(key, value: data))` returns a `NotificationResult`.
- Best when you need confirmation of receipt. Combine with `subscribe()` on the receiving side.

### 3. Notification / Event-driven Communication
- Receiver: `atClient.notificationService.subscribe(regex: "pattern", shouldDecrypt: true).listen((notif) {...})`.
- Sender: `notificationService.notify(NotificationParams.forUpdate(key, value: data))`.
- Lightweight, supports regex filtering, auto-reconnects on network loss.

### 4. RPC (Remote Procedure Call)
- Use built-in `AtRpc` / `AtRpcClient`.
- Construct `AtRpc` with `AtRpcCallbacks` (`handleRequest`, `handleResponse`) and an allowList.
- Requester: `rpc.sendRequest(toAtSign: target, request: AtRpcReq(...))`. Responses arrive via `handleResponse`.
- Multi-instance agents: immutable mutex race (`Metadata()..immutable = true`, `useRemoteAtServer = true`) so only one instance handles each request. Or `ServiceFactoryWithNoOpSyncService()` with per-session ephemeral storage for stateless scale-out.

### 5. Streaming Data
- Continuous updates: `subscribe(regex: "stream.*")` plus a `notify()` loop/timer on the sender.
- Bulk file transfer: encrypt locally, upload to a web file service, send URL+key as an encrypted AtKey.
- `atClient.startMonitor()` is DEPRECATED — always use `notificationService.subscribe()`.
- Pre-release streaming for LLM answers: https://pub.dev/packages/at_stream

### 6. TCP / Socket Communication (NoPorts)
- E2E encrypted TCP with zero open listening ports via `noports_core` (`Npt` class) or `npt` CLI.
- Creates a local tunnel port that routes through the Atsign Platform to a remote device.
- Docs: https://pub.dev/packages/noports_core and https://docs.noports.com/

## Data Storage and Synchronization
- `atClient.put()` creates an AtKey locally and auto-syncs to the atServer when online.
- NEVER store application data in local files. Use AtKeys so data flows across all of an owner's devices.
- Use namespaces to separate application data.
- Use `sharedWith` for targeted sharing; `metadata.isPublic = true` only for intentionally public data.

## Multi-Agent Coordination
Two recognized patterns:
1. **Mutex race with immutable keys**: first instance to write an immutable key (`Metadata()..immutable = true`) with `useRemoteAtServer = true` wins. Other instances see the key exists (the `put` returns failure) and skip. Use TTL to auto-release.
2. **ServiceFactoryWithNoOpSyncService**: per-session ephemeral storage with no sync. For stateless processing like request-response agents.

## Encrypted File Transfer
- Encrypt the file client-side (e.g. AES with a per-file key).
- Upload to a chosen web file service.
- Share the URL and the symmetric key as an encrypted AtKey with the recipient.
- Recipient downloads ciphertext and decrypts locally.

## Policy Enforcement
- Enforce access policies via the originating Atsign's sharing decisions (only share with the explicitly permitted Atsign).
- Use TTL on AtKeys for time-bounded grants.
- "Kill switches" can be implemented by deleting the shared AtKey or revoking the symmetric key used to encrypt referenced content.

## Error Handling
- Always wrap AtClient ops in try/catch.
- Common exceptions: `AtClientException`, `AtKeyException`, `AtConnectException`.
- Check `AtClientManager.getInstance().atClient.syncService.syncInProgress` before critical ops.

## Key Conventions
- Use namespaces matching the application (e.g. `cleanroom`).
- Prefer descriptive key names: `<purpose>_<analysisId>` e.g. `submission_a_<analysisId>`.
- For shared keys set `sharedWith` explicitly. For ephemeral keys set TTL/TTR/TTB as appropriate.
- For immutable mutex keys set `Metadata()..immutable = true` and `..ttl = <ms>`.
